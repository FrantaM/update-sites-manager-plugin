UpdateSites Manager plugin
==========================

アップデートサイトを管理するJenkinsプラグイン

これはなに？
------------

UpdateSites Manager は、管理画面に「アップデートサイトの管理」を追加する [Jenkins](http://jenkins-ci.org/) プラグインです。

Jenkinsが新しいプラグインを取得するサイト(アップデートサイト)は hudson.model.UpdateCenter.xml というファイル内に記載されます。
このプラグインは、アップデートサイトをJenkinsの管理画面から管理する機能を提供します:

* 「Jenkinsの管理」画面に新しく「アップデートサイトの管理」というメニューを追加します。
* Jenkinsに登録されているアップデートサイトの表示、追加、編集、削除を行えます。
* 新しいアップデートを追加する際は、以下の項目を指定可能です:
	* サイトを無効にする
		* 一時的にアップデートサイトを無効にする場合にチェックします。
	* ID
		* アップデートサイトのIDです。設定するべき値はアップデートサイト側から指定されます。
	* サイトURL
		* アップデートサイトのURLです。通常、update-center.jsonのURLになります。
	* メモ
		* Jenkinsでは使用しないので、管理上で分かりやすい自由な内容を記載してください。
	* CA証明書
		* このアップデートサイトで使用するCA証明書です。アップデートサイトが自己署名証明書で署名されている場合に、サイトから配布されるCA証明書を記載します。

新しいアップデートサイトを作成するには
-------------------------------------

[How to create your own Jenkins Update Center] (https://github.com/ikedam/backend-update-center2/wiki/How-to-create-your-own-Jenkins-Update-Center) を参照してください。


制限事項
--------

* 以下のアップデートサイトは「アップデートサイトの管理」に表示されますが、設定を変更したり、削除することができません。
	* デフォルトのアップデートサイト(IDが"default"のアップデートサイト)。このサイトの設定は「プラグインの管理」画面から行えます。
	* 他のプラグインによって作成されるアップデートサイト。各プラグインが提供する管理画面から設定してください。
